# ملفات بيانات مرجعية (تجريبية) — Monark 3D Mining

هذا المجلد فيه ملفات CSV جاهزة تقدر تستخدمها للتجربة على أي مشروع جديد. البيانات وهمية لكن هندسيًا متسقة (نفس البئر بين الملفات، أعماق منطقية، لا تداخلات).

| الملف | المحتوى | فين تريّه في الواجهة |
|---|---|---|
| `collar.csv` | مواقع 3 آبار (DDH-001..003) | زر **Import Drillholes (CSV)** → خانة **Collar CSV** |
| `survey.csv` | ميل واتجاه كل بئر (dip/azimuth) | نفس النافذة → خانة **Survey CSV** |
| `assay.csv` | نتائج تحليل الذهب لكل عمق، فيها أمثلة على `blank`/`duplicate`/`standard` وقيمة تحت حد الكشف (`<0.02`) | نفس النافذة → خانة **Assay CSV** |
| `lithology.csv` | نوع الصخر + RQD% + Core Recovery% | نفس النافذة → خانة **Lithology CSV** |
| `topography.csv` | شبكة نقاط ارتفاع (36 نقطة) حوالين منطقة الآبار | زر **Upload Topo / Trenches / Veins** → خانة **Topography** |
| `trench.csv` | 3 خنادق استكشافية بموقعها ودرجتها | نفس النافذة → خانة **Trenches** |

## ملاحظة عن عينة الـ QA/QC
ملف `assay.csv` فيه صف بمعيار مرجعي اسمه `STD_GOLD_01`. عشان تتفعّل مقارنة المدى المتوقع، لازم تضيف نفس الاسم ده في لوحة **QA/QC Reference Standards** (مثلاً: Min=1.4, Max=1.7, Unit=g/t) *قبل* رفع الملف — لو رفعته من غير ما تضيف المعيار هيتحط عليه علامة `unconfigured` مش خطأ يمنع الرفع.
